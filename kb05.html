Untuk Apa Manajemen I/O?

Salah satu tugas sistem operasi adalah mengatur dan mengontrol perangkat input/output (I/O) dan operasi I/O. Manajemen (I/O) tersebut sangat diperlukan karena perangkat I/O sebagai sarana user untuk bisa berkomunikasi dengan komputer. Misalnya seperti keyboard, mouse, speaker, disk drives, dan sebagainya. Manajemen I/O diperlukan agar user dapat langsung menggunakan perangkat I/O tanpa harus menginialisasi terlebih dahulu. Oleh karena itu, dalam setiap sistem operasi selalu terdapat manajemen I/O.

Adanya manajemen I/O memungkinkan perangkat I/O untuk bisa berkomunikasi dengan user. Di dalamnya terdapat I/O manager yang berfungsi menerjemahkan perintah read dan write menjadi read dan write IRP (I/O request packets) yang dilanjutkan oleh driver perangkat. Atau secara mudahnya, I/O manager menyediakan "device driver" yang umum sehingga operasi I/O dapat seragam (membuka, membaca, menulis, menutup). Misalnya user menggunakan operasi yang sama untuk membaca file pada hard-disk, CD-ROM, dan flash disk.

Beberapa fungsi sistem operasi dalam manajemen input/output yaitu:

  • Buffering (penyanggaan)
    Menampung data sementara dari/ke perangkat I/O.

  • Scheduling (Penjadwalan)
    Melakukan penjadwalan pemakaian I/O sistem supaya lebih efisien.

  • Spooling
    Meletakkan suatu pekerjaan program pada penyangga, agar setiap perangkat dapat mengaksesnya saat perangkat tersebut siap.

  • Menyediakan driver perangkat yang umum
    Driver digunakan agar sistem operasi dapat memberi perintah untuk melakukan operasi pada perangkat keras I/O yang umum, seperti optical drive, media penyimpanan sekunder, dan layar.

  • Menyediakan driver perangkat yang khusus
    Driver khusus digunakan agar sistem operasi dapat memberi perintah untuk melakukan operasi pada perangkat keras I/O tertentu, seperti kartu suara, kartu grafis, dan modem.

Manajemen I/O Linux

Pada Linux, semua perangkat input/output (I/O) terlihat seperti berkas, perangkat tersebut dapat dibuka, ditutup, dibaca, dan ditulis. Hal-hal itu dilakukan menggunakan system calls yang sama dan standar untuk memanipulasi berkas. Setiap perangkat pada sistem diwakilkan oleh sebuah file khusus perangkat, contohnya yaitu sebuah harddisk diwakilkan dengan sebuah file /dev/sda.

Semua perangkat yang dikontrol oleh driver device yang sama memiliki nomor device besar yang umum. Kemudian ada nomor device kecil yang digunakan untuk membedakan antara device-device yang berbeda dan pengontrol-pengontrolnya. Misalnya setiap partisi dalam suatu harddisk utama memiliki sebuah nomor device kecil yang berbeda yiatu /dev/sda1, /dev/sda2, dan seterusnya.

Linux membagi perangkat atau devices ke dalam tiga kelas yaitu:

  1. Block devices
    Terdiri dari perangkat-perangkat menyimpan, menerima, dan mengirim informasi dalam blok-blok berukuran tertentu. Ukurannya berkisar 128 sampai 1024 byte dan memiliki alamat tersendiri, sehingga memungkinkan user untuk membaca atau menulis blok-blok secara independen, yaitu dapat membaca atau menulis sembarang blok tanpa harus melewati blok-blok lain. Contoh perangkat blok yaitu disk, tape, CD ROM, dan optical disk. Fungsi block devices sendiri didukung oleh 2 sistem komponen, yaitu block buffer cache dan request manager. Block buffer cache bertugas sebagai pool dari buffer I/O aktif dan cache dari “completed I/O”. Request manager adalah lapisan software yang mengelola konten read dan write buffer dari dan menuju block-device driver.

  2. Character devices
    Perangkat yang menerima dan mengirimkan aliran karakter tanpa membentuk suatu struktur blok. Contohnya yaitu printer, mouse, dan perangkat yang bukan disk. Perangkat ini tidak memerlukan random access untuk sebuah block data. Akan tetapi, tiap perangkat telah memiliki berbagai fungsi yang telah terdaftar pada kernel yang diimplementasikan pada file operasi I/O.

  3. Network devices
    Bagian ini berbeda dengan block atau character devices, user harus berkomunikasi secara tidak langsung dengan cara membuka sambungan dengan subsistem jaringan kernel. Data device ini mengandung informasi tentang perangkat jaringan yaitu nama, informasi bus, informasi protokol, jenis, alamat, antrian paket, dan fungsi pendukung.

Rangkuman

Manajemen input/output (I/O) pada sistem operasi sangat diperlukan agar perangkat I/O sebagai sarana user untuk bisa berkomunikasi dengan komputer. Beberapa fungsi sistem operasi dalam manajemen input/output yaitu buffering, scheduling, spooling, menyediakan driver perangkat yang umum dan khusus.

Pada Linux, semua perangkat input/output (I/O) terlihat seperti berkas, perangkat tersebut dapat dibuka, ditutup, dibaca, dan ditulis. Linux membagi perangkat atau devices ke dalam tiga kelas yaitu block devices, character devices, dan betwork devices.
