Organisasi File

Linux memiliki sistem file menyerupai pepohonan (tree), yaitu dimulai dari root, kemudian direktori dan sub dirrektori. Sistem file pada Linux diatur secara hirarki, yaitu dimulai dari root dengan simbol “/”.

Kita dapat menciptakan file dan direktori mulai dari root sampai ke bawah. Direktori dapat berisi file-file dan direktori lagi (disebut juga subdirektori).

Direktori Standar

Setelah proses instalasi, Linux otomatis menciptakan direktori-direktori yang dibuat berdasarkan Filesystem Hierachy Standart (FHS). FHS merupakan seperangkat petunjuk untuk penempatan file dan direktori pada sistem operasi yang mirip UNIX. Tujuannya agar dapat mendukung penggunaan aplikasi, program administrasi sistem, pengembangan program, skrip, dan dapat menyatukan dokumentasi dari sistem.

Adapun direktori sesuai dengan FHS antara lain adalah sebagai berikut:

Direktori Deskripsi

/

Merupakan root folder, menduduki posisi puncakdalam hirarki. Direktori ini membawahi semua direktori, sehingga penulisan direktori lainnya menggunakan tanda / didepannya yang menunjukkan direktori tersebut dibawah root.

/bin

Berisi program binary dasar yang dibutuhkan oleh sistem maupun user.

/boot

Berisi program dan data yang dibutuhkan pada saat melakukan proses booting sistem.

/dev

Direktori tempat dimana device diproses.

/etc

Berisi file konfigurasi sistem.

/home

Direktori tempat menyimpan data user. Setiap user yang terdaftar secara otomatis akan dibuatkan direktori pada /home.

/lib

Berisi file-file library dari aplikasi yang terdapat pada sistem.

/media

Berisi media yang dapat dipasang dan dilepas pada komputer. Misal saat Anda memasang flash disk ke komputer, maka Anda bisa menemukan direktori flash disk di /media. Media lainnya yaitu CD/DVD ROM, floppy disk, harddsik eksternal, dsb.

/mnt

Direktori tempat pengaitan sistem sementara.

/opt

Berisi paket aplikasi tambahan yang diinstall ke dalam sistem.

/proc

Filesystem untuk menjalankan proses.

/root

Direktori home untuk user root.

/sbin

Berisi program biner yang dibutuhkan untuk menjalankan sistem. Aplikasi pada direktori ini biasanya hanya bisa dijalankan oleh administrator atau root.

/tmp

Direktori tempat menyimpan file yang bersifat sementara.
/usr

Berisi program-program yang bisa di akses oleh user. Dalam direktori ini ada sub direktori /usr/bin dan /usr/sbin yang menyimpan aplikasi executable yang fungsinya sama dengan file-file pada direktori /bin dan /sbin.

/var

Untuk menyimpan file-file yang isinya sering berubah, seperti database, user mail, dsb. Di dalamnya ada sub direktori /var/log yang berisi log files yang menyimpan informasi tentang aktivitas sistem.
Properti File

File pada Linux memiliki beberapa properti atau atribut. Beberapa diantaranya yaitu:

  • Tipe File yaitu dapat berupa:
Karakter Arti
- File biasa
d Direktori
l Symbolic link
b Block special file
c Character special file
s Socket link
p FIFO
  • Izin akses menentukan hak akses terhadap suatu file.
  • Jumlah link merupakan banyak link pada suatu file.
  • Pemilik menentukan siapa pemilik file.
  • Group menentukan group pemilik file.
  • Ukuran file berupa besar file yang dinyatakan dalam byte.
  • Waktu pembuatan berupa informasi kapan file terakhir dimodifikasi.
  • Nama file

Gambar 13.2 berikut adalah contoh properti suatu file yang dapat diperoleh dengan perintah ls -l nama_file

Symbolic Link

Sebelum membahas symbolic link, terdapat link yang merupakan sebuah cara untuk memberikan lebih dari satu nama file dengan data yang sama. Bila file asli dihapus, maka data yang baru juga terhapus. Adapaun formatnya yaitu:

ln file_asli file_duplikat

file_duplikat disebut hard link dimana kedua file akan muncul sama, mengakibatkan jumlah link atau link count  = 2). Jika salah satu file_asli atau file_duplikat dirubah, maka perubahan akan terjadi pada kedua file tersebut.

Symbolic link diperlukan bila suatu file dibuat “link” dengan direktori/file yang berada pada partisi yang berbeda. Tipe file menjadi l (link) dan file tersebut menunjuk ke file aslinya. Formatnya yaitu:

ln -s /FULLPATH/file_asli /FULLPATH/file_duplikat

Pilihan -s (shortcut) merupakan bentuk soft link dimana jumlah link count pada file asal tidak berubah. Pada bentuk soft link, symbolic link dapat dilakukan pada file yang tidak ada, sedangkan pada hard link tidak bisa. Perbedaan lain, symbolic link dapat dibentuk melalui media disk atau partisi yang berbeda, tetapi pada hard link terbatas pada partisi disk yang sama.

Melihat Tipe File

Suatu file dapat dilihat tipenya menggunakan format:

file fileName(s)

Tipe file akan ditampilkan seperti contoh berikut:

$ file halo.txt info.html
halo.txt: ASCII text
info.html: HTML document

Mencari File

Apabila ingin mencari file dapat digunakan perintah:

  • find
    Format: find directory -name targetfile -print
    Untuk mencari file yang bernama targetfile.

  • which
    Format: which command
    Untuk mengetahui letak system utility.

  • locate
    Format: locate string
    Untuk mencari file pada semua direktori dengan lebih cepat dan ditampilkan dengan path yang penuh.

Mencari Teks pada File

Untuk mencari teks pada file digunakan perintah grep (General RegularExpression Print) dengan format:

grep option pattern files

Grep akan mencari file sesuai nama dengan pattern yang diberikan dan akan menampilkan baris yang sesuai.

LATIHAN 1. DIREKTORI

  1. Melihat direktori Home.

    $ pwd

    $ echo $HOME

  2. Melihat direktori terkini dan parent direktori.

    $ pwd

    $ cd .

    $ pwd

    $ cd ..

    $ pwd

    $ cd

  3. Membuat satu direktori, lebih dari satu direktori, dan sub direktori.

    $ pwd

    $ mkdir A B C A/D A/E B/F A/D/A

    $ ls

    $ ls A

    $ ls A/D

  4. Menghapus direktori (satu atau lebih). Operasi ini hanya dapat dilakukan pada direktori kosong dan hanya dapat dihapus oleh pemiliknya, kecuali bila diberikan ijin aksesnya.

    $ rmdir B  (ada pesan error, mengapa?)

    $ ls B

    $ rmdir B/ B

    $ ls B  (ada pesan error, mengapa?)

  5. Navigasi direktori dengan instruksi cd untuk pindah dari satu direktori ke direktori lain.

    $ pwd

    $ ls

    $ cd A

    $ pwd

    $ cd ..

    $ pwd

    $ cd /home/<user>/C

    $ pwd

    $ cd /<user>/C  (ada pesan error, mengapa?)

    $ pwd

LATIHAN 2. MANIPULASI FILE

  1. Perintah cp untuk mengkopi file atau seluruh direktori.

    $ touch contoh

    $ cp contoh contoh1

    $ ls

    $ cp contoh A

    $ ls A

    $ cp contoh contoh1 A/D

    $ ls A/D

  2. Perintah mv untuk memindah file.

    $ cp contoh1 contoh2

    $ cp contoh1 contoh3

    $ ls

    $ mv contoh1 A

    $ ls A

    $ mv contoh2 contoh3 A/D

    $ ls A/D

  3. Perintah rm untuk menghapus file.

    $ cp contoh contoh4

    $ ls

    $ rm contoh4

    $ ls

    $ rm A  (ada pesan error, mengapa?)

    $ ls

    $ rm -rf A

    $ ls

LATIHAN 3. SYMBOLIC LINK

Membuat shortcut (file link)

$ echo "Halo apa kabar" > halo.txt

$ ls -l

$ ln halo.txt z

$ ls -l

$ cat z

$ mkdir folderKu

$ ln z folderKu/halo.juga

$ cat folderKu/halo.juga

$ ln -s z bye.txt

$ ls -l z bye.txt

$ cat bye.txt

LATIHAN 4. MELIHAT TIPE FILE

$ ls -l

$ file halo.txt

$ file bye.txt halo.txt

LATIHAN 5. MENCARI FILE

  1. Perintah find

    $ find /home -name "*.txt" -print > myerror.txt

    $ cat myerror.txt

    $ find . -name "*.txt" -exec wc -l '{}' ';'

  2. Perintah which

    $ which ls

  3. Perintah locate

    $ locate "*.txt"

LATIHAN 6. MENCARI TEKS PADA FILE

$ grep Halo .txt
