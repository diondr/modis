#18 Pencarian Kesalahan pada Hasil Instalasai Sistem Operasi

Tujuan / Uraian Materi / Lembar Kerja / Rangkuman



Tujuan

Pada pembelajaran ini Anda diharapkan:

  • Dapat menjelaskan pencarian kesalahan pada hasil instalasai Ubuntu.
  • Mampu melakukan pencarian kesalahan pada hasil instalasai Ubuntu.

Lanjut ke Materi »

Uraian Materi

Pembelajaran mengenai pencarian kesalahan pada hasil instalasai ini dimulai dengan pembahasan tentang kesalahan umum yang sering terjadi pada hasil instalasai. Kemudian prosedur pencarian kesalahan pada hasil instalasi. Setelah itu latihan yang berkaitan dengan bahasan tersebut siap Anda kerjakan. Selamat belajar :)

Kesalahan Umum

Kesalahan pada hasil instalasi Ubuntu bisa saja terjadi karena beberapa hal. Seperti dibahas pada pembelajaran sebelumnya jenis kerusakan atau kesalahan yang umum terjadi yaitu:

  • File installer (CD/DVD atau ISO) Ubuntu rusak, diakibatkan karena file tidak valid.

  • Proses instalasi yang lambat karena user mencentang pilihan Download updates while installing atau Install this third-party software pada tahap awal instalasi.

  • Sistem operasi lainnya terhapus, terjadi karena kesalahan dalam memilih tipe instalasi untuk pengaturan partisi harddisk.

  • Tidak bisa booting ke Ubuntu (Virtualalisasi) dikarenakan processor yang digunakan tidak mendukung kernel yang ada pada installer Ubuntu.

Prosedur Pencarian Kesalahan

Pencarian kesalahan pada hasil instalasi Ubuntu dapat dilakukan dengan beberapa cara, diantaranya yaitu:

  • Mengecek MD5
    Digunakan untuk mengecek integritas dan validitas dari suatu file.

  • Mengecek Defect Disk
    Untuk mengetahui apakah ada kerusakan pada CD/DVD installer yang digunakan.

  • Opsi instalasi
    Saat awal instalasi user mencentang pilihan Download updates while installing atau Install this third-party software yang mengakibatkan lambatnya proses instalasi.

  • Opsi tipe instalasi
    Kesalahan bisa terjadi pada tahap ini sehingga berakibat hilangnya sistem operasi yang sebelumnya terinstall pada komputer.

  • Pengaturan processor
    Dilakukan agar mendukung kernel yang digunakan saat instalasi secara virtual.



Lanjut ke Lembar Kerja »

Lembar Kerja

Alat dan Bahan
  • Komputer atau laptop
  • Koneksi internet
  • File installer Ubuntu.
  • File MD5 Ubuntu, Anda bisa mendapatkannya pada halaman UbuntuHashes (sesuaikan dengan versi file ISO Ubuntu yang Anda gunakan).

  • File installer winMd5Sum, Anda bisa mendapatkannya pada halaman winMd5Sum
Petunjuk
  • Awali setiap aktivitas dengan do'a, semoga mendapat kemudahan dan keberkahan.
  • Baca dan pahami langkah-langkah yang ada dengan seksama.
  • Pastikan komputer dalam keadaan baik dan benar.
  • Jangan meletakkan benda yang dapat mengeluarkan medan elektromagnetik (handphone, magnet, dan sejenisnya) di dekat komputer.
  • Gunakan komputer sesuai fungsinya dengan hati-hati.
  • Kerjakan secara baik, benar, dan sabar.
Latihan

LATIHAN 1. MENGECEK MD5 PADA UBUNTU

  1. Ubuntu telah menyediakan fasilitas untuk mengecek MD5 tanpa perlu menginstall aplikasi tambahan. Anda bisa melakukannya melalui terminal dengan perintah:

    $ md5sum <file_yang_dicek>

    Contoh: $ md5sum ubuntu-12.04.4-desktop-i386.iso

  2. Kemudian akan ditampilkan baris hasil md5sum.

  3. Bandingkan apakah sama dengan standar UbuntuHashes.



LATIHAN 2. MENGECEK MD5 PADA WINDOWS

  1. Sayangnya Windows tidak menyediakan secara langsung md5sum pada sistemnya. Anda harus menginstall aplikasi winMd5Sum yang telah Anda siapkan sebelumnya.

  2. Setelah winMd5Sum terinstall, buka aplikasi tersebut. Pilih file ISO Ubuntu pada bagian "File Name"

  3. Tunggu sampai proses selesai sehingga terlihat hasilnya pada bagian "MD5 Sum".

  4. Copy kode MD5 (sesuai versi Ubuntu yang dicek) dari UbuntuHashes ke bagian "Compare"

  5. Klik tombol Compare dan lihat hasilnya



LATIHAN 3. MENGECEK DEFECT DISK

  1. Pada saat Ubuntu Boot Screen dimulai, terlihat ada gambar keyboard dan orang di bagian bawah.
    Tekan sembarang tombol pada keyboard.

  2. Kemudian akan tampil Ubuntu Main Page Options seperti gambar di bawah ini.

  3. Pilih Check disc for defect dan tekan Enter.

  4. Maka proses pengecekan akan berjalan.

  5. Akan ditampilkan hasil pengecekan. Tekan sembarang tombol untuk merestart sistem.



LATIHAN 4. OPSI INSTALASI

  1. Jika saat instalasi Ubuntu berjalan lambat sekali, pertama matikan koneksi internet pada komputer Anda.

  2. Pada langkah persiapan instalasi, hilangkan centang pada pilihan Download updates while installing atau Install this third-party software.

  3. Kemudian Anda bisa melanjutkan proses instalasi.



LATIHAN 5. TIPE INSTALASI

  1. Jika pada hasil instalasi Ubuntu ternyata Anda tidak menemukan sistem operasi lain pada komputer Anda. Kemungkinan besar Anda salah memilih tipe instalasi atau salah mengatur partisi.

  2. Pada langkah Installation type pastikan pilihan Anda pada Something else.

  3. Tentukan partisi yang akan Anda gunakan sebagai sistem Ubuntu. Kemudian atur partisi tersebut untuk sistem Linux.

  4. Kemudian Anda bisa melanjutkan ke langkah instalasi selanjutnya.


Tugas

  1. Ada sebuah kesalahan yaitu user tidak bisa booting saat instalasi ubuntu secara virtual menggunakan VirtualBox seperti tampilan di bawah ini.

  2. Lakukan pengaturan pada komputer virtual Anda. Pilih System, kemudian pilih Processor. Centang pada bagian Enable PAE/NX.

  3. Booting ulang pada komputer virtual Anda.

  4. Apakah sekarang Anda bisa booting ke Ubuntu? Analisa mengapa hal tersebut bisa terjadi!

  5. Jika sudah selesai, Anda bisa menuliskan atau menceritakan apa saja yang Anda peroleh pada pembelajaran kali ini. Kemudian publikasikan tulisan Anda tersebut pada blog, website, atau media sosial.

Rangkuman

Kesalahan pada hasil instalasi Ubuntu bisa saja terjadi karena beberapa hal, diantaranya yaitu file installer (CD/DVD atau ISO) Ubuntu rusak, proses instalasi yang lambat, sistem operasi lainnya terhapus, dan tidak bisa booting ke Ubuntu pada instalasi secara virtual.
